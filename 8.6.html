4D-Rotor Gaussian Splatting: Towards Efficient Novel View Synthesis for Dynamic Scenes

这篇文章与4DGS类似，在3DGS的基础上加入了时间维度，在每个时间戳上对4D高斯体进行时间切片，生成动态的3D高斯体。这些3D高斯体可以无缝地投影到二维图像上。

时间切片是指将四维（包括时间）的高斯体在每个时间点上切开，得到一系列三维高斯体。这些三维高斯体可以分别表示动态场景在每个时间点的状态。然后，这些三维高斯体可以进一步处理和渲染成二维图像。这样做的好处是可以捕捉动态场景的变化，并在不同时间点上分别优化和渲染这些变化。

这个公式当 oi 趋近于 1 时，这一项趋向于 0；当 oi 趋近于 0 时，这一项趋向于正无穷大。这里的 oi 表示高斯球的透明度，使高斯球趋近于完全不透明，能去除一些伪影，同时高斯球透明度为 0 会被优化机制删除。

这个公式的作用：

在给定时间 t 对4D高斯进行切片时，会产生一个速度项 s 。因此，对于第 i 个高斯点，从其邻域空间Ωi中收集 K 个最近的4D点，并使它们的运动一致

  • 运动平滑性：它确保在时间维度上，物体的运动是平滑和一致的，避免出现突然的、不连续的运动变化。
  • 动态一致性：它使得在连续时间点上的高斯体的形状和位置变化保持一致，避免出现明显的伪影或不自然的变形。

  • 这周做了改善，直接使用简单的3D HasGrid编码器对高斯球进行多尺度编码，同时使用一个很简单的编码器对每一帧的id（也就是时间t）进行编码，然后送入3层MLP进行解码，得到高斯属性的偏移量。
  • 这周对颜色网络也做了修改，考虑了一下颜色网络和渲染图密切相关，所以在颜色网络中也要体现时间依赖性，将上述得到的时空特征也输入到颜色网络中，得到在渲染结果上的连续性。
  • 加了一个正则化项，灵感来自于上面阅读的论文，因为上面的论文没有公开代码，所以自己写了一下，大概表述为如下公式：
(1)L=−oilog⁡(oi)−(1−oi)log⁡(1−oi)
  • −oilog⁡(oi)：当 (o_i) 趋近于 1 时，这一项趋向于 0；当 (o_i) 趋近于 0 时，这一项趋向于正无穷大。
  • −(1−oi)log⁡(1−oi)：当 (o_i) 趋近于 0 时，这一项趋向于 0；当 (o_i) 趋近于 1 时，这一项趋向于正无穷大。

总体来说，模型的透明度输出接近于 0 或 1 时给予更低的损失值，从而驱动模型学习出明确的透明和不透明区域。
